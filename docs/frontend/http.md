# HTTP

## HTTP1.1

- 效率改进
  - TCP长连接
  - 浏览器为每个域名限制连接数（一般为6个）
  - CDN域名分片

- 缺陷
  - HTTP 队头阻塞
  - 带宽竞争
  - TCP 慢启动

## HTTP2

- 效率改进
  - 多路复用，一个域名使用一个 HTTP 长连接，资源并行请求，可以设置优先级
  - 二进制分帧，提高数据传输和解析的效率
  - HTTP 头部压缩
  - 服务器主动推送

- 缺陷
  - TCP 队头阻塞（丢包问题）
  - TCP 慢启动

## HTTP3

- 效率改进
  - QUIC：基于 UDP 的可靠传输协议
